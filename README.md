# 🏛️ 盛和塾运营管理系统

> 一个基于 Streamlit 的学员管理与运营分析平台，帮助盛和塾运营人员**可视化、数据化、智能化**地管理学员、分析活动、发现洞察。

---

## 📸 系统截图

| 总览看板 | 学员画像 | 综合洞察 |
|:--------:|:--------:|:--------:|
| 📊 指标卡 + 图表 | 👤 雷达图 + 评分 | 💡 AI 运营建议 |

---

## ✨ 核心功能

### 1. 📊 总览看板
- 学员总数、分中心数、班级数、活动记录数一目了然
- 各活动数据量柱状图
- 学员分层分布饼图（核心层 / 稳定层 / 潜力层 / 待激活 / 流失风险）
- 月度参与趋势折线图
- 分中心横向对比

### 2. 👤 学员分析
- **学员画像**：查看单个学员的详细信息、综合评分、雷达图
- **四维评分体系**：
  - 📋 活动参与度（30分）— 各类活动出勤率
  - 📚 阅读投入度（20分）— 打卡连续性、分享质量
  - 📈 近期趋势（25分）— 近3个月参与变化趋势
  - 🎯 参与广度（25分）— 活动类型多样性
- **五层分层体系**：
  | 分层 | 分数范围 | 说明 |
  |------|---------|------|
  | 🌟 核心层 | ≥85分 | 明星学员，积极参与各类活动 |
  | 💪 稳定层 | 70-84分 | 中坚力量，参与稳定 |
  | 🌱 潜力层 | 50-69分 | 有潜力，需针对性引导 |
  | ⚠️ 待激活 | 30-49分 | 参与度偏低，需要关怀 |
  | 🔴 流失风险 | <30分 | 长期未参与，面临流失 |
- **排行榜**：综合排行、出勤率排行、各维度排行
- **学员列表**：可按分层筛选查看

### 3. 📅 活动分析
- 7 类活动数据统计（小组学习会、班级学习会、课程、报告会、游学、读书打卡、读书分享）
- 各活动的月度参与趋势
- 活动参与率分析

### 4. 💡 综合洞察
- AI 智能生成运营建议（一键生成）
- 建议优先级：🔴 高 / 🟡 中 / 🟢 低
- 建议类型：学员个体 / 分中心 / 全系统
- 建议采纳跟踪

### 5. 📥 数据导入
- 支持 **Excel (.xlsx)** 和 **CSV (.csv)** 格式
- 支持 **8 类数据**导入：
  - 学员基本信息
  - 小组学习会记录
  - 班级学习会记录
  - 课程参与记录
  - 报告会参与记录
  - 游学参与记录
  - 读书打卡记录
  - 读书分享记录
- 列名**中英文自动映射**（如"姓名"→"name"）
- 导入日志可追溯
- 可下载导入模板

## 🚀 快速开始（小白版）

### 第一步：安装 Python

> 如果你还没有 Python，请先安装：
> 1. 打开 [python.org](https://www.python.org/downloads/)
> 2. 下载 **Python 3.9 ~ 3.12** 版本
> 3. 安装时**勾选** ✅ "Add Python to PATH"
> 4. 安装完成后，打开命令行（CMD）输入 `python --version`，看到版本号即成功

### 第二步：下载项目

```bash
# 下载项目（或直接从 GitHub 下载 ZIP 解压）
git clone https://github.com/spring-chao/seiwajyuku_system.git
cd seiwajyuku_system
```

### 第三步：安装依赖

```bash
pip install -r requirements.txt
```

> ⏳ 等待安装完成（约 1-3 分钟）

### 第四步：启动系统

```bash
streamlit run app.py
```

> 浏览器会自动打开 `http://localhost:8501`，看到系统界面即启动成功！🎉

### 第五步：导入数据

1. 点击左侧导航栏 **「📥 数据导入」**
2. 先下载模板 → 填写数据 → 上传文件
3. **先导入「学员基本信息」**，再导入其他活动数据
4. 完成后回到 **「📊 总览看板」** 查看数据

## 📖 数据导入指南

### 支持的数据类型

| 数据类型 | 必填字段 | 说明 |
|---------|---------|------|
| 学员基本信息 | 姓名(name) | 必须先导入 |
| 小组学习会记录 | 姓名, 日期 | 需学员已存在 |
| 班级学习会记录 | 姓名, 日期 | 需学员已存在 |
| 课程参与记录 | 姓名, 课程名, 课程日期 | 需学员已存在 |
| 报告会参与记录 | 姓名, 报告会名, 日期 | 需学员已存在 |
| 游学参与记录 | 姓名, 游学地, 日期 | 需学员已存在 |
| 读书打卡记录 | 姓名, 日期 | 需学员已存在 |
| 读书分享记录 | 姓名, 日期, 书名 | 需学员已存在 |

### 注意事项

1. ✅ **先导入学员信息**，再导入活动数据
2. ✅ 列名支持中英文，如 `姓名` = `name`，`日期` = `session_date`
3. ✅ 日期格式建议：`YYYY-MM-DD`（如 2024-01-15）
4. ✅ 系统内已含示例数据生成器：`sample_data/sample_data_generator.py`

## 🤖 智能运营建议

系统会根据数据自动生成运营建议，覆盖以下场景：

- 🔴 **高优先级**：分中心参与率低预警、学员长期未参与警告、分层结构优化
- 🟡 **中优先级**：阅读参与度提升、潜力学员激活、活动类型单一提醒
- 🟢 **低优先级**：进步表彰、风采展示推荐、经验分享建议

> 点击侧边栏 **「🤖 生成运营建议」** 按钮即可一键生成

## ❓ 常见问题

### Q: 启动后页面空白 / 报错？
A: 确认依赖已安装：`pip install -r requirements.txt`，然后重新启动。

### Q: 数据存在哪里？
A: 所有数据存储在项目目录下的 `seiwajyuku.db` 文件中，无需额外配置数据库。

### Q: 如何重置数据？
A: 删除 `seiwajyuku.db` 文件，重启系统即可自动重建空白数据库。

### Q: 支持多人同时使用吗？
A: 当前为单机版，适合运营人员在本地使用。多人协作请自行部署到服务器。
